【GitHub 高分开源项目】
### 板块一：【GitHub 高分开源项目】
━━━━━━━━━━━━━━━━━━━━

#### AI编程/Claude Code相关
1. anthropics/claude-plugins-official (Star数参考官方项目)
核心功能：Anthropic官方发布的Claude插件开发教程与示例集合
适用场景：开发Claude Desktop插件或集成Claude API的开发者
优势：官方维护，代码质量高，提供完整插件示例与最佳实践
不足：需要TypeScript基础，插件仅适用于Claude Desktop
地址：https://github.com/anthropics/claude-plugins-official
[用途] 快速学会为Claude Desktop编写插件，扩展Claude能力（如文件操作、API调用）
[步骤] 1. git clone仓库 2. 进入example目录 3. 运行npm install && npm run dev 加载到Claude Desktop

2. microsoft/azure-devops-mcp (Star数参考)
核心功能：Azure DevOps对接MCP（Model Context Protocol）的Server实现
适用场景：通过Claude Code或MCP客户端直接管理Azure DevOps工作项、PR、流水线
优势：微软官方项目，与Azure DevOps深度集成，支持工作项CRUD和CI/CD触发
不足：依赖Azure DevOps服务，需配置认证
地址：https://github.com/microsoft/azure-devops-mcp
[用途] 用Claude自然语言指令操作Azure DevOps（如“创建Bug并分配给张三”），提升项目管理效率
[步骤] 1. 安装Node.js 2. npx @microsoft/azure-devops-mcp 3. 在Claude Code MCP配置中添加该Server

#### AI/ML工具
3. agentgateway/agentgateway (Star数参考)
核心功能：统一的AI Agent网关，支持多模型路由、负载均衡、错误重试
适用场景：生产环境部署多个LLM（如GPT-4、Claude、Llama）时做流量管理和容灾
优势：轻量级Rust实现，延迟低，支持OpenAI API协议，可无缝替换
不足：社区较新，文档尚不完善
地址：https://github.com/agentgateway/agentgateway
[用途] 解决多模型调用时单点故障和成本控制问题，自动切换备用模型
[步骤] 1. 下载二进制文件 2. 编写配置文件指定模型 endpoints 3. 启动后替换原有API base URL

4. vercel-labs/agent-browser (Star数参考)
核心功能：为AI Agent提供浏览器自动化的Rust工具库，可截图、提取页面内容
适用场景：构建能访问网页的Agent（如自动抓取数据、填写表单）
优势：基于Playwright，支持无头浏览器，输出结构化结果，适合集成到Agent框架
不足：需要安装Chromium等依赖
地址：https://github.com/vercel-labs/agent-browser
[用途] 让Agent具备浏览网页能力，实现“帮我查一下某网站的最新价格”等功能
[步骤] 1. cargo add agent-browser 2. 调用Browser::new().navigate(url).extract_text()

#### 开发者工具
5. envoyproxy/ai-gateway (Star数参考)
核心功能：Envoy扩展的AI网关，支持请求限流、缓存、认证、监控
适用场景：企业级AI API统一入口，保护后端LLM服务，降低成本
优势：基于生产级Envoy代理，性能强，支持OpenAI和Anthropic协议\的最简洁配置
不足：需要了解Envoy配置语法，部署有一定门槛
地址：https://github.com/envoyproxy/ai-gateway
[用途] 为AI API提供安全网关，防止滥用、控制预算、记录审计日志
[步骤] 1. 下载最新release 2. 修改配置中的上游URL和API密钥 3. docker run -v ./config.yaml:/etc/envoy/envoy.yaml envoyproxy/ai-gateway

### 板块二：【前沿大模型动态】
━━━━━━━━━━━━━━━━━━━━

1. OpenAI推理模型推翻80年几何猜想
核心特点：o3模型自主证明了Erdos单位距离猜想（平面单位距离色数问题），AI首次独立解决长期开放的数学猜想
优势：推理能力突破，证明过程可复现验证
不足：仅解决特定几何问题，通用推理能力仍需提升，证明细节尚未完全公开
调用方式：仅通过OpenAI API（o3模型）
免费渠道：无免费渠道，API付费使用
链接：https://www.zhihu.com/question/1838391418396852000（知乎讨论，含原始论文链接）
[用途] 验证AI在数学研究中的辅助价值，可尝试用o3模型探索自己领域中的开放问题
[步骤] 1. 获得OpenAI API key 2. 使用o3模型（如用ChatGPT选择o3） 3. 输入具体数学或科学问题，要求给出证明思路

2. 腾讯混元翻译模型Hy-MT2开源
核心特点：多语言翻译模型，支持100+语言，基于MoE架构，翻译质量达到商业级
优势：开源可商用，支持声音翻译（TTS），模型大小仅3B，可在消费级GPU运行
不足：需要自行部署推理服务，不支持图像翻译
调用方式：本地部署（开源）
免费渠道：完全开源，需自行部署
链接：https://www.oschina.net/news/352027/tencent-hunyuan-hy-mt2
[用途] 搭建私有翻译服务，处理多语种文档、字幕、网站内容翻译，无需调用外部API
[步骤] 1. git clone https://github.com/Tencent/Hunyuan-HyMT2 2. pip install -r requirements.txt 3. python run.py --model HyMT2 --text "你好世界"

3. 字节开源统一视频/图像生成理解编辑模型Lance
核心特点：单一模型同时支持文生图、文生视频、图像理解、视频理解、图像编辑等多项功能
优势：统一架构，减少部署多模型成本；支持任意尺寸和分辨率；理解与生成互相增强
不足：需要大量计算资源（建议A100 80G）；视频生成分辨率目前最高720p
调用方式：本地部署（开源）
免费渠道：开源模型，需自行部署
链接：https://www.oschina.net/news/352066/bytedance-lance
[用途] 用一个模型完成“根据文字生成图片/视频-理解内容-修改编辑”全流程，适合内容创作和自动化生产线
[步骤] 1. 下载模型权重（约20GB） 2. 使用transformers库加载 3. 调用pipe("a cat")生成图像，或pipe("describe this image")进行理解

### 板块三：【免费 AI 工具】
━━━━━━━━━━━━━━━━━━━━

1. Google搜索AI Agent改版
主打功能：Google搜索集成“信息Agent”，能理解复杂查询并逐步生成报告，类比Perplexity的深度研究
使用门槛：需要美国IP，访问google.com，部分用户已灰度开放
价格：免费（需Google账号）
实用玩法：输入“对比特斯拉Model 3和比亚迪汉，从价格、续航、充电速度三个维度分析”，Agent自动搜索并生成表格报告
适合人群：研究者、学生、任何需要快速获取结构化信息的人
链接：https://www.oschina.net/news/352071/google-search-ai-agent
[用途] 替代传统手动搜索，对复杂问题直接获取综合答案，节省大量浏览时间
[步骤] 1. 登录Google账号 2. 在搜索框输入问题（建议中文） 3. 点击搜索结果页顶部的“生成报告”按钮（若未显示则暂未灰度到）

2. Spotify AI播客问答与简报生成
主打功能：播客收听时实时提问（如“刚才提到的研究是哪一年”），收听后自动生成精华摘要
使用门槛：Spotify Premium订阅（付费），支持英文播客
价格：付费订阅（约$10.99/月）
实用玩法：收听编程播客时，直接问“这个库的安装命令是什么？”AI从播客内容中提取答案并显示
适合人群：播客重度听众，特别是学习类、新闻类播客用户
链接：https://techcrunch.com/2026/05/21/spotify-adds-ai-powered-q-and-a-and-briefing-generation-features-to-podcasts/
[用途] 提高播客信息获取效率，不必回听，随时问细节
[步骤] 1. 在Spotify打开任意播客 2. 点击播放/暂停按钮旁的“AI”图标 3. 输入问题或点击“生成摘要”

3. YouTube Shorts AI混音/重混功能
主打功能：基于他人YouTube Shorts视频，用AI生成自己的变体（换背景、换声音、加动作）
使用门槛：YouTube账号（免费），需在移动App内操作
价格：免费
实用玩法：找到热门舞蹈Short，点击“Remix”按钮，选择“AI变体”，系统自动生成不同背景和动作的视频，可发布到自己的频道
适合人群：短视频创作者、想要快速生产内容的人
链接：https://www.theverge.com/news/2026/5/21/youtube-shorts-ai-remix
[用途] 无需自己拍摄和剪辑，利用AI对已有视频进行再创作，快速产出差异化内容
[步骤] 1. 打开YouTube App，打开任意Short 2. 点击右下角“Remix”按钮 3. 选择“AI Remix”，调整参数（时长、风格） 4. 发布

要点：面向金融行业的低代码AI Agent构建平台，内置反欺诈、风控、合规等预置Agent，支持自然语言配置业务流程
如何落地：金融机构可调用平台API快速搭建智能客服、智能投顾、贷后管理等Agent，无需从零训练模型
链接：https://www.36kr.com/p/3304995161410817
[用途] 金融科技公司或企业内部IT团队，可用该平台在1周内上线一个智能咨询或风控Agent，代替传统规则引擎
[步骤] 1. 进入“点金”控制台 2. 从模板库选择“智能客服Agent” 3. 配置业务对接API（如账户查询、风控规则） 4. 上线测试

2. VAPD AgentKit：可组合Agent前端通用库
要点：开源的前端库（VAPD AgentKit），提供可复用的UI组件与状态管理，快速搭建类似Perplexity或ChatGPT的多Agent交互界面
如何落地：开发者直接复用组件（如消息流、模型选择器、知识检索面板），减少UI开发工作量

3. 腾讯操作系统级AI助手Marvis
要点：集成在Windows/Mac/Linux桌面系统中的AI助手，支持全局呼出（Alt+Space），可控制本地应用（打开文件、系统设置）、截图提问、跨应用搜索
如何落地：用户安装后获得类似Apple Intelligence的全局AI能力，开发者可调用其API集成到自己的应用
链接：https://www.oschina.net/news/352059/tencent-marvis
[用途] 用自然语言操作电脑（如“把昨天的会议纪要放到D盘”，AI自动完成），对比已有的微软Copilot，Marvis目前免费测试
[步骤] 1. 下载Marvis安装包 2. 安装后按Alt+Space唤出 3. 输入“打开Chrome并搜索AI落地案例”或“帮我截图并翻译”

### 板块六：【全栈开发实操】
━━━━━━━━━━━━━━━━━━━━

1. TimescaleDB 2.27.1 发布（时序数据库）
技术点：新增连续聚合支持动态时间桶（减少存储与查询优化），修复Hypertable压缩导致的分区溢出问题
避坑要点：升级时需注意旧版本的自定义压缩策略可能不兼容，需先禁用压缩再升级；连续聚合若使用自定义时间桶函数，需重新创建或调整
链接：https://www.oschina.net/news/352058/timescaledb-2-27-1
[用途] 用于存储IoT传感器数据、金融时间序列、应用监控指标，升级后可降低50%存储成本并提升查询速度
[步骤] 1. docker pull timescale/timescaledb:2.27.1-pg16 2. 停止旧容器（确保数据卷挂载） 3. 启动新版本容器，执行ALTER EXTENSION timescaledb UPDATE

2. JeeSite V5.17.1 发布（Spring Boot/Vue快速开发平台）
技术点：集成Ollama本地模型接口，支持AI代码生成（从数据库表生成增删改查前后端代码）；修复Vue3组件热更新卡顿问题
避坑要点：若使用JeeSite自带的代码生成AI功能，需本地安装Ollama并下载codegeex模型（约7GB），默认未打开，需手动配置application.yml
链接：https://www.oschina.net/news/352010/jeesite-v5-17-1
[用途] 快速生成Java+Vue全栈CRUD代码，配合AI模板减少重复编码，适合接外包或内部管理系统开发
[步骤] 1. 下载JeeSite最新版 2. 在application.yml中开启gen.ai.enabled=true，配置Ollama地址 3. 设计数据库表，运行代码生成向导，选择AI优化”即可生成包含前后端的完整模块
